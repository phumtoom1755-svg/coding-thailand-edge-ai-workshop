## 1. โจทย์ของทีม

Smart lighting (1-2 ประโยค บอกว่าจะแก้ปัญหาอะไรในชีวิตจริง):

```
แก้ปัญหาเปลืองค่าไฟโดยไม่จำเป็น ด้วยระบบตั้งเวลาปิด (Timer) หมดปัญหาการเดินคลำหาสวิตช์มืดๆ หรือต้องลุกจากเตียงไปปิดไฟ ด้วยคำสั่งเสียงและการทำงานอัตโนมัติเมื่อเดินเข้าห้อง สามารถตั้งค่าเปิด-ปิดไฟตามเวลาเสมือนมีคนอยู่บ้าน ผู้จัดทำได้คิดแนวทางการเปลี่ยนระบบแสงสว่างแบบเดิมให้สามารถควบคุมได้จากทุกที่เพื่อมอบความสะดวกสบายสูงสุด ช่วยประหยัดค่าไฟจากการลดการเปิดทิ้งไว้และปรับความสว่างตามจริง เพิ่มความปลอดภัยให้บ้านด้วยระบบเปิด-ปิดอัตโนมัติเพื่อป้องกันขโมยและลดอุบัติเหตุยามค่ำคืน พร้อมทั้งส่งเสริมสุขภาพผู้อยู่อาศัยด้วยการปรับแสงให้สอดคล้องกับนาฬิกาชีวิตและสร้างบรรยากาศที่เหมาะสมกับทุกกิจกรรม
```

## 2. Class Definition 
   
ระบุ class ที่จะใช้ + เกณฑ์ที่ทำให้แต่ละ class แตกต่าง:

| # | ชื่อ Class | คำอธิบาย (3-4 ประโยค) | สิ่งที่ทำให้ต่างจาก class อื่น |
|---|---|---|---|
| 1 |SmartLight| คลาสที่เป็นตัวแทนของหลอดไฟหรืออุปกรณ์ส่องสว่างอัจฉริยะทุกดวงในบ้าน ทำหน้าที่รับคำสั่งโดยตรงเพื่อปรับเปลี่ยนสถานะทางกายภาพของแสง | ตัวกำหนดแสงไฟ |
| 2 |SmartSwitch|คลาสสวิตช์อัจฉริยะที่ทำหน้าที่เป็นตัวกลางเชื่อมต่อระหว่างการกดปุ่มทางกายภาพกับระบบเน็ตเวิร์ก คลาสนี้ช่วยให้อุปกรณ์สวิตช์แบบเดิมสามารถส่งสัญญาณดิจิทัลไปสั่งการหลอดไฟอัจฉริยะข้ามห้องได้| ตัวประมวลข้อมูลเข้าเพื่อส่งไปสั่งคำสั่งคลาส SmartLight |
| 3 |Sensor|คลาสสำหรับอุปกรณ์ตรวจจับสภาพแวดล้อมเพื่อส่งข้อมูลดิบเข้ามาประมวลผลในระบบอัตโนมัติ ทำหน้าที่คอยเฝ้าระวังการเปลี่ยนแปลง |ตัวรับข้อมูลเข้าจากการกระทำของผู้ใช้ | 

**ตัวอย่าง (Track A, Fall Detector):**
| # | ชื่อ Class | คำอธิบาย | สิ่งที่ทำให้ต่าง |
|---|---|---|---|
| 1 | ยืน | ผู้ใช้ยืนนิ่ง หรือยืนเฉยๆ ใส่ที่เอว | accel แกน Z สูง, gyro ต่ำ |
| 2 | เดิน | เดินปกติ ก้าวเท้าสลับ | accel oscillation, periodic |
| 3 | ล้ม | ล้มลงพื้นกะทันหัน | high accel spike + horizontal Z |

## 3. Edge Cases — สิ่งที่อาจสับสน

ระบุสถานการณ์ที่อาจทำให้โมเดลสับสน:

```
1. หากมีเสียงรบกวนมากๆ อาจทำให้เกิดการเข้าใจผิดได้ต่อการเปิดปิดไฟได้

2. หากพูดเบาอาจทำให้ระบบไม่สามารถจับเสียงได้

3. ห้องที่มีเสียงสะท้อนมากๆ อาจทำให้ระบบเข้าใจผิดได้
```

**วางแผนแก้:**
```
อัดเสียงเทรนโมเดลโดยใช้สภาพแวดล้อมต่างๆ สถานการณ์ต่างๆ เช่น เสียงรบกวนใช้วิธีจับจุดโฟกัสของเสียง เสียงสะท้อนหน่วงเวลาเปิดปิด หรือ อัดในสภาพแวดล้อมนั้น 
```

## 5. Diversity Plan — กันอคติ (Bias)

ตอบ 3 คำถาม:

### a) มีหลาย "environment" ไหม?
```
ห้องที่มีเสียงรบกวน ห้องปกติ
```

### b) มีหลาย "subject" ไหม? (กรณี Vision/Audio/Motion)
```
เก็บ 3 เสียงจากคนในทีม ในลักษณะพื้นที่ที่มีเสียงรบกวน และ ห้องปกติ
```

### c) มีหลาย "variation" ไหม?
```
มีหลายระดับเสียง ความเร็วในการพูด และ ระยะการพูด
```
