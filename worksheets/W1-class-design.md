## 2. Class Definition 
1   
ระบุ class ที่จะใช้ + เกณฑ์ที่ทำให้แต่ละ class แตกต่าง:

| # | ชื่อ Class | คำอธิบาย (3-4 ประโยค) | สิ่งที่ทำให้ต่างจาก class อื่น |
|---|---|---|---|
| 1 |Normal Posture |เป็นท่านั่งปกติที่ผู้ใช้นั่งหลังตรง ศีรษะอยู่ในระดับเหมาะสม  |ศีรษะไม่ก้ม หลังไม่ค่อม และแขนอยู่ในท่าปกติ
| 2 |Neck Bending |เป็นท่าทางที่ผู้ใช้ก้มคอลงมากเกินไปขณะใช้งานคอมพิวเตอร์หรือโทรศัพท์| มีการก้มศีรษะลงอย่างชัดเจน มุมคอเอียงต่ำกว่าปกติ|
| 3 |Hunchback / Arm Twist |เป็นท่านั่งที่ผู้ใช้นั่งหลังค่อมหรือวางแขนผิดท่า เช่น บิดข้อมือหรือยกไหล่มากเกินไป ทำให้เกิดแรงกดบริเวณหลัง ไหล่ และข้อมือ ซึ่งเสี่ยงต่อการปวดเมื่อยสะสม |ลักษณะหลังโค้ง ไหล่ห่อ หรือแขนและข้อมืออยู่ในตำแหน่งผิดธรรมชาติ | 

**ตัวอย่าง (Track A, Fall Detector):**

| # | ชื่อ Class | คำอธิบาย | สิ่งที่ทำให้ต่าง |
|---|---|---|---|
| 1 | Normal Posture | ผู้ใช้นั่งในท่าปกติ หลังตรง ศีรษะอยู่ระดับเหมาะสม และแขนวางตามธรรมชาติ | ศีรษะตรง หลังไม่ค่อม มุมคอปกติ |
| 2 | Neck Bending | ผู้ใช้ก้มคอลงขณะใช้งานคอมพิวเตอร์หรือโทรศัพท์เป็นเวลานาน | ศีรษะเอียงลง มุมคอต่ำกว่าปกติ |
| 3 | Hunchback / Arm Twist | ผู้ใช้นั่งหลังค่อมหรือวางแขนผิดท่า เช่น บิดข้อมือหรือยกไหล่ | high accel spike + horizontal Z |

## 5. Diversity Plan — กันอคติ (Bias)

ตอบ 3 คำถาม:

### a) มีหลาย "environment" ไหม?
```
[เก็บในห้องเรียน + ห้องอื่น? แสงสว่าง/แสงน้อย? เวลาต่างกัน?]
```

### b) มีหลาย "subject" ไหม? (กรณี Vision/Audio/Motion)
```
[เก็บจากหลายคนในทีม? ลักษณะ background ต่างกัน?]
```

### c) มีหลาย "variation" ไหม?
