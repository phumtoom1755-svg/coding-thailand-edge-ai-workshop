## 1. โจทย์ของทีม

Smart lighting (1-2 ประโยค บอกว่าจะแก้ปัญหาอะไรในชีวิตจริง):

```
แก้ปัญหาเปลืองค่าไฟโดยไม่จำเป็น ด้วยระบบตั้งเวลาปิด (Timer) หมดปัญหาการเดินคลำหาสวิตช์มืดๆ หรือต้องลุกจากเตียงไปปิดไฟ ด้วยคำสั่งเสียงและการทำงานอัตโนมัติเมื่อเดินเข้าห้อง สามารถตั้งค่าเปิด-ปิดไฟตามเวลาเสมือนมีคนอยู่บ้าน
     ผู้จัดทำได้คิดแนวทางการเปลี่ยนระบบแสงสว่างแบบเดิมให้สามารถควบคุมได้จากทุกที่เพื่อมอบความสะดวกสบายสูงสุด ช่วยประหยัดค่าไฟจากการลดการเปิดทิ้งไว้และปรับความสว่างตามจริง เพิ่มความปลอดภัยให้บ้านด้วยระบบเปิด-ปิดอัตโนมัติเพื่อป้องกันขโมยและลดอุบัติเหตุยามค่ำคืน พร้อมทั้งส่งเสริมสุขภาพผู้อยู่อาศัยด้วยการปรับแสงให้สอดคล้องกับนาฬิกาชีวิตและสร้างบรรยากาศที่เหมาะสมกับทุกกิจกรรม
```

## 2. Class Definition 
1   
ระบุ class ที่จะใช้ + เกณฑ์ที่ทำให้แต่ละ class แตกต่าง:

| # | ชื่อ Class | คำอธิบาย (3-4 ประโยค) | สิ่งที่ทำให้ต่างจาก class อื่น |
|---|---|---|---|
| 1 | |เป็นท่านั่งปกติที่ผู้ใช้นั่งหลังตรง ศีรษะอยู่ในระดับเหมาะสม  |ศีรษะไม่ก้ม หลังไม่ค่อม และแขนอยู่ในท่าปกติ
| 2 |Neck Bending |เป็นท่าทางที่ผู้ใช้ก้มคอลงมากเกินไปขณะใช้งานคอมพิวเตอร์หรือโทรศัพท์| มีการก้มศีรษะลงอย่างชัดเจน มุมคอเอียงต่ำกว่าปกติ|
| 3 |Hunchback / Arm Twist |เป็นท่านั่งที่ผู้ใช้นั่งหลังค่อมหรือวางแขนผิดท่า เช่น บิดข้อมือหรือยกไหล่มากเกินไป ทำให้เกิดแรงกดบริเวณหลัง ไหล่ และข้อมือ ซึ่งเสี่ยงต่อการปวดเมื่อยสะสม |ลักษณะหลังโค้ง ไหล่ห่อ หรือแขนและข้อมืออยู่ในตำแหน่งผิดธรรมชาติ | 

**ตัวอย่าง (Track A, Fall Detector):**

| # | ชื่อ Class | คำอธิบาย | สิ่งที่ทำให้ต่าง |
|---|---|---|---|
| 1 | Normal Posture | ผู้ใช้นั่งในท่าปกติ หลังตรง ศีรษะอยู่ระดับเหมาะสม และแขนวางตามธรรมชาติ | ศีรษะตรง หลังไม่ค่อม มุมคอปกติ |
| 2 | Neck Bending | ผู้ใช้ก้มคอลงขณะใช้งานคอมพิวเตอร์หรือโทรศัพท์เป็นเวลานาน | ศีรษะเอียงลง มุมคอต่ำกว่าปกติ |
| 3 | Hunchback / Arm Twist | ผู้ใช้นั่งหลังค่อมหรือวางแขนผิดท่า เช่น บิดข้อมือหรือยกไหล่ | high accel spike + horizontal Z |

## 3. Edge Cases — สิ่งที่อาจสับสน

ระบุสถานการณ์ที่อาจทำให้โมเดลสับสน:

```
1. หากมีเสียงรบกวนมากๆ อาจทำให้เกิดการเข้าใจผิดได้ต่อการเปิดปิดไฟได้

2. หากพูดเบาอาจทำให้ระบบไม่สามารถจับเสียงได้

3. ห้องที่มีเสียงสะท้อนมากๆ อาจทำให้ระบบเข้าใจผิดได้
```

**วางแผนแก้:**
```
อัดเสียงเทรนโมเดลโดยใช้สภาพแวดล้อมต่างๆ สถานการณ์ต่างๆ เช่น เสียงรบกวนใช้วิธีจับจุดโฟกัสของเสียง เสียงสะท้อนหน่วงเวลาเปิดปิด หรือ อัดในสภาพแวดล้อมนั้น
```

## 5. Diversity Plan — กันอคติ (Bias)

ตอบ 3 คำถาม:

### a) มีหลาย "environment" ไหม?
```
[เก็บในห้องเรียน + ห้องอื่น? แสงสว่าง/แสงน้อย? เวลาต่างกัน?]
```

### b) มีหลาย "subject" ไหม? (กรณี Vision/Audio/Motion)
```
[เก็บจากหลายคนในทีม? ลักษณะ background ต่างกัน?]
```

### c) มีหลาย "variation" ไหม?
```
[มุม/ระยะ/ความเร็ว/ระดับเสียง ที่แตกต่างกัน?]
```
